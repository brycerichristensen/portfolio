Flight Configuration

When this engine propels a rocket to 12,000 feet this Spring, it will become the first successful SRAD hybrid engine in the state of Utah. I am responsible for developing all of the controls and electronics both on board the rocket and on the ground. In order to feel confident that the engine would perform correctly, we strapped the engine to the launch rail and performed a series of hot fire tests. You can watch one of these tests below!

Software Architecture

I developed a comprehensive Finite State Machine (FSM) implemented in C++. This FSM is run simultaneously on three devices- the onboard flight computer, the oxidizer fill trailer, and the ground station. Each device is set to transition into energetic states only if it has an active connection to the other devices. Complex safety and control logic was implemented to manage critical transitions during fueling and launch.

Custom PCB Design

I designed a custom printed circuit board (PCB) to interface the ESP32 flight computer with our sensor suite. The PCB contains connection points for four pressure transducers, a thermocouple amplifier, a servomotor, and an optional load cell. It provides voltage regulation, signal processing, and status LEDs, ensuring the system is controlled safely and data is accurately processed.

Propulsion Control Unit (PCU)

Using OnShape, I CAD-modeled an avionics sled for the PCU. The sled was designed to precisely house each electronic component, including the PCB, thermocouple shield, load cell amplifier, UBEC, and two batteries. Attention was given to tolerances to ensure each component fit snugly.

Integrated Assembly

This CAD model shows the flight-ready propulsion configuration. I designed a bracket that integrates the PCU onto threaded rods while ensuring that the control system did not interfere with any fluid components or structural components. It also had to be designed so that a 6-inch diameter body tube could fit around it. I worked closely with the fluids and structures teams to meet all necessary design requirements.

Physical Assembly

Once the design was finalized, the PCB was milled at a shop on campus. I soldered each component on, taking great care to avoid shorts or damaging sensitive components. I then 3D-printed the sled and integrated each part into the assembly./p>.

Fill Trailer Control

To properly fill the flight tank with nitrous oxide prior to launch, a control system was designed for the oxidizer fill trailer (OFT). These controls commmunicate via radio with the ground station and PCU, actuating solenoid values at appropriate times to safely fill the tank remotely.

Ground Station UI

In order to monitor the status of each device, I designed a user interface using Python to display the status of each valve and current data readings from each sensor. It provides warnings and connectivity status to enable a safe launch.

Burst Testing

To ensure structural integrity of our SRAD flight tank, we performed a destructive hydrostatic burst test. FEA showed that the tank would burst at 4000 psi (a factor of safety of 4), and the tank actually burst at 4050 psi. These results validated our FEA and gave us confidence in future tanks.

Hot Fire Campaign

The culmination of these efforts resulted in a successful hot fire campaign to validate each subsystem. The control system worked as intended, safely filling, igniting, and then depressurizing the hybrid engine configuration.
